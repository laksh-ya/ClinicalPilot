Final Architecture

Clinical Decision Support,
Redefined.

A complete multi-agent clinical decision support system. Three specialist agents debate through an adversarial Critic, backed by live PubMed citations, parallel drug safety checks, and Groq-powered conversational AI.

Data Flow Summary

FHIR/EHR/Text → Anonymizer → Unified Schema →
Clinical → [ Literature || Safety ] →
Critic (per round, 2-3 max) →
Synthesizer → Validator → SOAP Note + Human Loop

The Orchestrator Controller (GPT-4o or MedGemma) feeds the Unified Context into three simultaneous expert agents.

🧠

Clinical Agent

GPT-4o / MedGemma-2-9b

Input / Tools

Unified context + Few-shot chain-of-thought prompting.

Generates Output
  • Differential dx (ranked)
  • Risk scores
  • Initial SOAP draft

GPT-4o-mini (Fast)

Input / Tools

  • BioPython Entrez (parallel queries)
  • LanceDB RAG (infrastructure ready)

  • Confidence levels
  • Contradictions

Safety Agent

GPT-4o + FDA/DrugBank

Input / Tools

  • DrugBank CSV (drug name validation)
  • openFDA API (parallel label lookups)

  • Severity tiers
  • Contraindications
ℹ️

MedGemma Node: Run via Ollama. Use 9B for speed. Fallback to OpenAI if local inference fails. Configured in config.py as USE_LOCAL_LLM=true.

4 / Protocol

Debate Layer (Critic + Rounds)

Async Python with structured state management. Fixed 2-3 rounds max for deterministic timeout limits avoiding infinite loops. Med Error Panel runs in parallel via asyncio.gather.

Round 1
( Initialization )

Clinical agent generates first. Then Literature + Safety run in parallel using its output.

★
Each Round
( Critic Review )

Critic runs after every round — reviews all outputs and logs dissent constraints.

Round 3
( Final Revision )

Agents revise. If consensus fails, task heavily flagged for human review.

⚖️

The Critic Agent Constraints

During Round 2, the Critic strictly reviews the aggregated context against three primary vectors to produce a dissent_log:

📑
EHR Contradictions

"Ddx includes Pulmonary Embolism, but no D-dimer was ordered in lab history."

🔬
Evidence Gaps

"Literature agent found no recent robust papers on this specific rare presentation."

⚠️
Safety Misses

"Safety agent missed renal dosing adjustment requirement for this patient's current GFR."

Guardrails Framework

Non-Negotiable

Powered by Pydantic v2 + Custom Validation Rules

  • ✓ Output schema validation strictly to SOAP structure
  • ✓ Zero hallucinated medications permitted
  • ✓ Differential logic must produce ≥ 2 options
  • ✓ Safety flags must be explicitly called out if triggered

Observability Stack

LangSmith

  • - Trace every agent call
  • - Token usage per agent computation
  • - Latency breakdown metrics per layer
  • - Visual debate round mapping
🔄
Human-in-the-Loop Process

When an output hits the UI, the Doctor can heavily edit the generated SOAP. This feeds directly back to the Critic Agent to trigger a new debate round (hard-capped at 1 single iteration loop to prevent endless cycles).

7 / Confidence Module

Medical Uncertainty Quantification (MUC)

Optional but highly recommended visual logic. Appends explicit certainty scoring to Clinical Agent outputs directly in the UI.

High Confidence

Multiple independent guidelines agree entirely. Accompanied by strong, irrefutable evidence bases.

Medium Confidence

Guidelines differ slightly on approach, or available literature features limited primary evidence.

Low Confidence

Highlights a rare presentation or relies on completely contradictory literature. Demands intense review.

Emergency Mode

Always Accessible

A dedicated screen and big red button. Designed specifically for paramedics in the field, triage nurses, and on-call physicians reacting at 3am.

Text / Voice Input Only No EHR upload in emergency path. Speed is the point.
Bypasses Debate + Anonymizer Orchestrator routes directly to Clinical + Safety agents in parallel. No PHI scrubbing (speed tradeoff).
Sub-5 Second Target Returns top 3 differentials, immediate red flags, ESI score, and next call-to-action.
Emergency System Prompt Dedicated triage prompt focused on ESI scoring and time-critical conditions. Same GPT-4o model.

AI Chat Assistant

4 / Protocol

Debate Layer (Critic + Rounds)

Orchestrated with LangGraph. Fixed 2-3 rounds max for deterministic timeout limits avoiding infinite loops.

Round 1
( Initialization )

Clinical, Literature, Safety agents generate output independently.

★
Round 2
( Critic Review )

Reviews all outputs and explicitly logs dissent constraints.

Round 3
( Final Revision )

Agents revise. If consensus fails, task heavily flagged for human review.

⚖️

The Critic Agent Constraints

During Round 2, the Critic strictly reviews the aggregated context against three primary vectors to produce a dissent_log:

📑
EHR Contradictions

"Ddx includes Pulmonary Embolism, but no D-dimer was ordered in lab history."

🔬
Evidence Gaps

"Literature agent found no recent robust papers on this specific rare presentation."

⚠️
Safety Misses

"Safety agent missed renal dosing adjustment requirement for this patient's current GFR."

5 & 6 / Output & Protection

Validation, Guardrails & Observability

Final Synthesizer & Validator

Takes debate logs and agent outputs, generating terminal files after strict final logic passes.

Validator Sub-Checks
  • Output Completeness Verification
  • Format Compliance checks
  • Safety Clearance validation
  • MUC Uncertainty Flags applied correctly
SOAP Note Export Human Loop Gate Full Traced Logs

Guardrails Framework

Non-Negotiable

Powered by Guardrails AI / Pydantic

  • ✓ Output schema validation strictly to SOAP structure
  • ✓ Zero hallucinated medications permitted
  • ✓ Differential logic must produce ≥ 2 options
  • ✓ Safety flags must be explicitly called out if triggered

Observability Stack

LangSmith / Langfuse / Phoenix

  • - Trace every agent call
  • - Token usage per agent computation
  • - Latency breakdown metrics per layer
  • - Visual debate round mapping
🔄
Human-in-the-Loop Process

When an output hits the UI, the Doctor can heavily edit the generated SOAP. This feeds directly back to the Critic Agent to trigger a new debate round (hard-capped at 1 single iteration loop to prevent endless cycles).

7 / Confidence Module

Medical Uncertainty Quantification (MUC)

Optional but highly recommended visual logic. Appends explicit certainty scoring to Clinical Agent outputs directly in the UI.

High Confidence

Multiple independent guidelines agree entirely. Accompanied by strong, irrefutable evidence bases.

Medium Confidence

Guidelines differ slightly on approach, or available literature features limited primary evidence.

Low Confidence

Highlights a rare presentation or relies on completely contradictory literature. Demands intense review.

Emergency Mode

Always Accessible

A dedicated screen and big red button. Designed specifically for paramedics in the field, triage nurses, and on-call physicians reacting at 3am.

Voice / Text Input Only Absolutely no EHR upload allowed (speed is the ultimate point).
Bypasses Debate Layer Orchestrator routes directly to Clinical + Safety agents in parallel.
Sub-5 Second Output Returns top 3 differentials, immediate red flags, and next call-to-action.
Tuned Processing Exclusively trained on emergency triage logic (ESI scoring, time-critical conditions).

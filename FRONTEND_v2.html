Final Architecture — System Design

ClinicalPilot

A multi-agent AI system for clinical decision support — combining EHR parsing, literature retrieval, safety checks, and adversarial debate to assist physicians at the point of care.

GPT-4o / MedGemma LangGraph HL7 FHIR Microsoft Presidio ChromaDB DrugBank PubMed API LangSmith
01 — System Overview

End-to-End Data Flow

ClinicalPilot ingests multi-modal clinical data, anonymizes PHI, runs parallel specialist agents, debates their outputs adversarially, and delivers a validated SOAP note with full observability.

Layer 1 — Input Gateway

Multi-Modal Ingestion

Three simultaneous input channels converge into a single processing gateway before any downstream handling.

FHIR API
HL7 FHIR v4 JSON — Patient, Condition, Observation, MedReq resources
EHR Upload
PDF / CSV documents — Unstructured.io with PyPDF2 fallback
Free Text / Voice
Clinical prompts, voice transcription, STT integration
Layer 1b — PHI Scrubbing

Anonymizer — Microsoft Presidio

All data passes through PHI scrubbing before touching any LLM. Presidio uses FHIR-specific entity recognizers to strip patient-identifiable information while preserving full clinical meaning. Synthetic data fallback via Gretel.ai. Zero PHI leaves the anonymization gate.

Layer 2 — Processing & Normalization

Unified Schema: PatientContext

Three parallel parsers (FHIR → JSON chunks, EHR → PDF text, Text → voice transcription) all normalize into a single PatientContext schema containing demographics, conditions, medications, labs/vitals, and the current clinical prompt.

Layer 3 — Agent Layer

Three Parallel Specialist Agents

The Orchestrator fans out to three specialist agents that run concurrently via asyncio.gather(). Each agent has distinct tools, prompting strategies, and output schemas.

Layer 4 — Debate Layer

Critic-Driven Adversarial Review

A Debate Orchestrator runs 2–3 fixed rounds. Round 1: independent outputs. Round 2: Critic Agent challenges all outputs against EHR contradictions, evidence gaps, and safety misses. Round 3: revised consensus. Unresolved conflicts flagged for human review.

Layer 5 — Validation & Output

Final Synthesizer + Validator Agent

Checks output completeness, format compliance, safety clearance, and MUC confidence thresholds. Produces three simultaneous outputs: a structured SOAP Note, a Human-in-the-Loop gate for doctor edits and re-debate, and full observability logs tracing every agent decision.

02 — Agent Layer

Three Specialist Agents

Each agent runs in parallel with distinct prompting strategies, external tool integrations, and structured output schemas. The Orchestrator coordinates them via a LangGraph state machine.

🧠

Clinical Agent

GPT-4o / MedGemma-2-9b
  • Few-shot chain-of-thought prompting
  • Unified PatientContext as input
  • Ranked differential diagnoses
  • Clinical risk scores
  • Initial SOAP note draft
Outputs
Differentials (ranked) · Risk Scores · SOAP Draft
📚

Literature Agent

GPT-4o-mini (fast)
  • PubMed E-utilities search
  • Europe PMC API retrieval
  • Internal RAG (ChromaDB)
  • Evidence snippet extraction
  • Contradiction detection
Outputs
Evidence Snippets · Confidence Levels · Contradictions
🛡️

Safety Agent

GPT-4o / Rule-Based
  • DrugBank open data (CSV)
  • RxNorm (NLM) lookups
  • FDA API cross-checks
  • Drug interaction detection
  • Renal/hepatic dosing flags
Outputs
Structured Warnings · Severity Ratings · Contraindications
⚙️

Orchestrator

GPT-4o primary with MedGemma-2-9b (Ollama/vLLM) as local fallback. Runs all three agents in parallel via asyncio.gather(). State managed through LangGraph. If local GPU inference fails, auto-falls back to OpenAI API.

03 — Debate Layer

Adversarial Critic Rounds

A structured debate forces agents to reconcile conflicting evidence. Fixed 2–3 rounds prevent infinite loops. Human re-trigger adds one additional iteration maximum.

⚔️

Debate Orchestrator — LangGraph State Machine

Max 2–3 Rounds
Round 01
Independent Generation
  • Clinical Agent generates differentials + risk scores
  • Literature Agent retrieves evidence snippets
  • Safety Agent flags interactions and warnings
  • All outputs stored in shared LangGraph state
→
Round 02
Critic Agent Review
  • Challenges Ddx vs. EHR data contradictions
  • Flags evidence quality gaps in literature
  • Catches safety misses (e.g., renal dosing errors)
  • Outputs structured dissent_log
→
Round 03
Revised Consensus
  • Agents revise outputs based on critique
  • Unresolved conflicts flagged for human review
  • Doctor edit triggers 1 additional round (max)
  • Final state passed to Validator
Critic Agent Logic
# Critic challenges each agent against cross-cutting concerns critic_checks = { "ehr_contradictions": "Ddx includes PE but no D-dimer ordered?", "evidence_gaps": "No recent papers on this rare presentation?", "safety_misses": "Renal dosing adjustment missed for patient GFR?", "consensus_check": "All agents agree → pass | disagree → flag human" } # dissent_log tracks every challenged claim with source agent dissent_log = [{ "claim": ..., "challenger": "critic", "severity": ... }]

05 — Supplementary Systems

Extra Features

Dedicated subsystems that extend ClinicalPilot beyond core diagnostics — emergency triage, medication safety, and imaging classification.

🚨

Emergency Mode

A dedicated screen with a big red button, always accessible. Designed for speed — voice or text input only, no EHR upload. Bypasses the debate layer entirely.

  • Voice or text input only — no EHR upload (speed is the point)
  • Bypasses debate layer — Orchestrator goes direct to Clinical + Safety agents in parallel
  • Output in under 5 seconds: top 3 differentials, immediate red flags, call-to-action
  • Tuned specifically on emergency triage logic (ESI scoring, time-critical conditions)
  • Designed for paramedics in field, triage nurses, on-call physicians at 3 AM
How it works
Orchestrator receives urgent input → skips debate layer → fans out to Clinical Agent + Safety Agent only → returns top 3 differentials with red flags in <5s. No literature lookup, no consensus rounds. Speed over completeness.
💊

Medical Error Prevention Panel

Runs as a persistent sidebar on every case, not just drug-specific queries. Automatically extracts medications from the prompt or EHR and runs comprehensive safety checks.

Drug-Drug Interactions

Severity tiers: contraindicated, major, moderate. Each flag shows the problem, mechanism in plain language, and recommendation.

Drug-Disease Contraindications

NSAIDs in CKD, metformin in contrast dye cases, and other condition-specific flags caught automatically.

Dosing Alerts

Renal and hepatic adjustment needed — flags when patient labs indicate impaired clearance pathways.

Population Flags

Pregnancy, pediatric, and elderly population-specific warnings surfaced automatically from context.

Output format per flag
Each flag card displays: the problem (what's wrong), the mechanism (explained in plain language a clinician can quickly verify), and the recommendation (what to do about it).
🫁

Imaging Classifier — Bonus Demo Feature

Upload a chest X-ray or CT scan image to a CNN-based classifier that feeds results directly into the EHR Agent's context, influencing the differential diagnosis.

  • Upload chest X-ray or CT scan images directly through the interface
  • CNN-based classifier (pre-trained model — no training from scratch required)
  • Detects: pneumonia, pleural effusion, cardiomegaly, pneumothorax
  • Results feed into the Clinical Agent's context → influences differential diagnosis
  • Visual AI that's actually integrated into the pipeline, not bolted on
Integration point
Image upload → pre-trained CNN classifier → structured findings (e.g., "bilateral infiltrates suggestive of pneumonia, confidence 0.87") → appended to PatientContext → Clinical Agent incorporates into differential ranking.

07 — Uncertainty Quantification

Medical Uncertainty Scoring

Every differential diagnosis carries a confidence tier displayed in the UI. Low-certainty outputs are automatically flagged for physician review.

🟢

High Confidence

Multiple clinical guidelines agree. Strong, reproducible evidence base. Safe to present as primary recommendation.

🟡

Medium Confidence

Guidelines differ across sources, or evidence base is limited. Physician should review supporting literature before acting.

🔴

Low Confidence

Rare clinical presentation, contradictory literature, or novel drug interaction. Mandatory human-in-the-loop review triggered.

08 — Build Priority

1-Day Build Order

Prioritized by clinical impact and liability protection. Each step builds on the last. The debate layer is the differentiator — ship it early.

1

Input Gateway + Anonymizer

Presidio integration. FHIR endpoint + PDF upload. All PHI scrubbed before LLM contact. Non-negotiable foundation.

2

Clinical Agent + Few-Shot Prompting

Biggest immediate impact. PatientContext → GPT-4o → ranked differentials + SOAP draft. Chain-of-thought examples critical.

3

Debate Layer + Critic Agent

The "wow factor" differentiator. LangGraph state machine. 2–3 fixed rounds. dissent_log captures all challenges.

4

Safety Agent + DrugBank

Liability protection. DrugBank CSV + RxNorm. Structured severity-rated warnings. Renal/hepatic dosing adjustments.

5

Literature Agent + PubMed RAG

BioPython Entrez + ChromaDB retrieval. Europe PMC fallback. Evidence snippets with confidence levels.

6

Observability — LangSmith Tracing

Trace every agent call. Token usage per agent. Latency breakdown. Debate round visualization. Free tier sufficient for dev.

7

Human-in-the-Loop UI

Doctor edits SOAP → feeds back to Critic Agent → triggers max 1 additional debate round. Prevents infinite revision loops.

8

MUC Confidence Scoring

🟢/🟡/🔴 per differential. Threshold-based auto-flagging. Displayed prominently in the physician UI.
